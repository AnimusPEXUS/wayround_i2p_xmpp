If driving, then look for event == 'ssl wrapped' and then:

  1. restart input machine with driven socket streamer, input event and objects hubs;
  2. restart output machine with driven streamer and output event hub
  3. wait till machines working
  4. restart stream with start_stream command

drive(obj)[source]¶

Drives to STARTTLS, basing on obj.

If obj.tag is {http://etherx.jabber.org/streams}features and it is contains {urn:ietf:params:xml:ns:xmpp-tls}starttls element, then:

  1. switch self.status to 'requesting tls'
  2. run _start()
  3. start STARTTLS sequence sending starttls element
  4. wait while self._driving == True
  5. return self._result
Return type:str

Return can be one of following values:

value meaning
‘no tls’ TLS not proposed by server
‘success’ TLS layer engaged
‘stream stopped’ stream was closed by server
‘stream error’ some stream error encountered
‘failure’ server returned {urn:ietf:params:xml:ns:xmpp-tls}failure
‘response error’ wrong server response
‘programming error’ if you received this - mail me a bug report
